# **🚀 CursorHolmes**

> 목격자의 음성·텍스트 진술을 실시간 분석해, 실제 3D 지도 위에 사건 현장·인물·타임라인을 자동 재구성하는 **초기 수사 단계 지원 도구**입니다.

## **💡 어떤 동기에서 프로젝트를 시작하게 되었나요? (어떤 문제를 해결하려고 하나요?)**

사건 초기 단계에서 수사관은 여러 목격자의 **구두 진술**을 받지만, 이를 **공간·시간·인물 관계**로 정리하는 과정은 여전히 수기 메모와 상상에 의존하는 경우가 많습니다. 진술 간 모순, 위치 관계 오해, 인상착의 정보 누락이 초기 수사 지연으로 이어질 수 있습니다.

CursorHolmes는 **“진술을 들은 즉시, 지도 위에서 사건이 어떻게 벌어졌는지 눈으로 확인할 수 있게”** 만드는 것을 목표로 시작했습니다. AI와 3D 지도를 결합해, 수사관이 현장을 빠르게 공유·검토하고 다음 수사 방향을 잡을 수 있도록 돕습니다.

---

## **👥 누구를 위한 서비스인가요?**

- **1차 사용자:** 사건 초기 단계에서 목격자 진술을 접수·정리하는 **경찰 수사관**
- **2차 수혜자:** 진술 내용을 시각적으로 공유받아 판단하는 **수사팀·지휘관**
- **사용 환경:** 민감 정보 보호를 위해 **수사관 PC 로컬 환경**에서 실행 (인터넷 공개 배포 불필요)

---

## **✨ 기존 방식보다 어떤 점이 나아지나요?**

| **기존 방식** | **CursorHolmes** |
| --- | --- |
| 진술을 텍스트/녹취로만 기록 | **실제 지도·3D 건물 위**에 인물·사건 흐름 즉시 재현 |
| 위치·이동 경로를 말로만 설명 | 용의자·피해자 **피규어 자동 배치**, 드래그로 위치 보정 |
| 인상착의를 별도 메모 | 진술에서 **상의·하의 색상 자동 추출** → 3D 몽타주 생성 |
| 여러 진술의 시간 순서 정리 어려움 | **타임라인 자동 생성**, 클릭 시 해당 장면 재현 |
| 보고서 작성에 시간 소요 | AI 기반 **수사 보고서 초안** 자동 생성 (키워드 폴백 지원) |

## **🎯 반드시 구현할 핵심 기능은?**

1. **진술 → 3D 장면 자동 변환** — 텍스트/음성 입력 시 용의자·피해자·소품을 지도 위에 배치
2. **실제 지도 기반 현장 재구성** — Mapbox 3D 건물 + Three.js 커스텀 레이어
3. **용의자 몽타주 3D 뷰어** — 인상착의 추출 및 정·측·후면 뷰
4. **사건 타임라인** — 진술 순서 기록 및 장면 재현
5. **사건 저장·이어하기** — 수사관별 사건 목록 관리 (localStorage)
6. **AI 수사 보고서** — Claude API 연동 + 오프라인 키워드 폴백

---

## **⚙️ 현재 실제로 동작하는 부분은?**

- ✅ 수사관 로그인 · 사건 생성·선택·저장 (localStorage)
- ✅ 지명/좌표 검색 → Mapbox 3D 지도 이동 (서울 AI 허브 등)
- ✅ 텍스트 진술 입력 → 키워드 분석 기반 장면 자동 생성
- ✅ Claude API 연동 시 AI 기반 정밀 장면 분석 (API 키 선택)
- ✅ Web Speech API 음성 진술 입력
- ✅ 3D 피규어 직접 배치 · 드래그 이동 · 더블클릭 삭제
- ✅ 용의자 몽타주 (색상·의류 파싱, 뷰 전환, 자동 회전)
- ✅ 진술 로그 · 타임라인 · 클릭 시 장면 재현
- ✅ AI 수사 보고서 생성 (API 없을 때 키워드 폴백)
- ✅ 서울 Open API **CCTV 위치** 지도 오버레이
- ✅ 진술 언급 **증거물** 3D 마커 (칼, 혈흔, 휴대폰 등)
- ✅ 오프라인 **데모 시나리오 프리셋** 재생
- ✅ 현장 캡처 PNG 저장

## **🧗 이 프로젝트를 하면서 부딪쳤던 도전과제들**

- **Mapbox + Three.js 좌표계 통합** — 지도 lng/lat와 3D 미터 좌표를 맞추는 커스텀 레이어 구현
- **자연어 진술 → 구조화된 장면 JSON** — AI 응답 파싱 실패 시 키워드 폴백 이중 구조 설계
- **인상착의 파싱** — “검은 반팔 상의, 초록 하의”처럼 **상·하의 색상을 문맥으로 구분** 추출
- **수동 배치 vs AI 재분석 충돌** — 수사관이 직접 옮긴 피규어가 AI 재분석 시 덮어씌워지지 않도록 `manual` 플래그 처리
- **경찰용 도구 vs 웹 배포** — 민감 정보 특성상 **로컬 실행** 중심 아키텍처로 방향 확정

---

## **🏆 가장 자랑스러운 부분**

- *“진술 한 줄을 넣으면, 실제 3D 지도 위에서 사건이 펼쳐진다”**는 end-to-end 경험을 순수 프론트엔드만으로 구현한 점입니다. API 키 없이도 키워드 모드로 데모가 가능하고, API가 있으면 AI 분석·보고서까지 확장됩니다. 수사관 워크플로(로그인 → 사건 → 진술 → 몽타주 → 타임라인 → 보고서)가 하나의 화면 안에서 자연스럽게 이어집니다.

---

## **📝 배운 점**

- GIS(지도)와 3D 그래픽스를 하나의 UX로 묶는 방법
- LLM 출력을 **신뢰할 수 있는 UI 상태**로 변환하려면 파싱·폴백·수동 보정 UX가 필수
- 공공기관·수사 도구는 **기능만큼 데이터 거버넌스**(로컬 저장, 배포 여부)가 중요
- 프레임워크 없이도 모듈화·상태 관리로 충분히 복잡한 앱을 만들 수 있음
